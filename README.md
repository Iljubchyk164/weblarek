# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-PresenterGalleryCatalog), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
PresenterGalleryCatalog - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

# Данные

## Слой коммуникации

### Класс AppApi

Хранит интерфейс для композиции.

Конструктор класса принимает экземпляр класса.

Поля класса:  
`api: IApi` - хранит интерфейс.

Методы класса:  
`get(): Promise<IProductResponse>` - получение объекта товаров;
`post(order: IOrder): Promise<IOrdersResponse>` - отправка заказа на сервер;

## Типы

`TPayment` = 'card' | 'cash' | '';
`TBuyerErrors` = Partial<Record<keyof IBuyer, string>> - объект с необязательными полями ('payment' | 'email' | 'phone' | 'address')

## Интерфейсы 

`IProduct` - интерфейс товара, содержит поля: `id` (идентификатор), `description` (описание), `image` (URL изображения), `title` (название), `category` (категория), `price` (цена, может быть null)

`IBuyer` - интерфейс данных покупателя, содержит поля: `payment` (способ оплаты), `email` (электронная почта), `phone` (номер телефона), `address` (адрес доставки)

`IOrder` -Расширяет IBuyer. Представляет полный заказ. `total` (Общая сумма заказа), `items` (Список идентификаторов товаров)

`IOrdersResponse` - Ответ со списком заказов. `id` (Идентификатор заказа), `total` (Общая сумма заказа)

`IProductResponse` - `total` (Общее количество товаров), `items` (Массив товаров)

### Класс Presenter
Главный контроллер приложения, отвечающий за координацию взаимодействия между моделями, представлениями и API. Реализует паттерн MVP (Model-View-Presenter).

Конструктор класса:
  `constructor(config: IPresenterConfig)` - принимает конфигурационный объект со всеми зависимостями.

Параметры конфигурации:

  `api: АppApi` - экземпляр API для взаимодействия с сервером

  `event: EventEmitter` - экземпляр EventEmitter для управления событиями

  `models` - объект с моделями данных:
  {
  `product: Product` - модель товаров

  `cart: Cart` - модель корзины

  `customer: Customer` - модель покупателя
  }

  `views` - объект с представлениями:
  {
  `gallery: GalleryView` - представление галереи

  `modal: ModalView` - представление модального окна

  `header: HeaderView` - представление шапки

  `success: SuccessView` - представление успешного заказа

  `cart: CartView` - представление корзины

  `cardPreview: CardPreview` - представление карточки в превью

  `formContacts: FormContacts | null` - форма контактов

  `formOrder: FormOrder | null` - форма заказа
  }

  `templates` - объект с HTML-шаблонами

  `containers` - объект с контейнерами DOM

Поля класса
  `config	IPresenterConfig`	Конфигурация приложения со всеми зависимостями

Публичные методы
  `init(): Promise<void>` - Инициализирует приложение. Загружает товары с сервера, рендерит их в галерею и настраивает обработчики событий.

Приватные методы
  `setupEventListeners(): void` - Настраивает все обработчики событий приложения. Подписывается на события, генерируемые представлениями, и связывает их с соответствующими методами.

Обрабатываемые события:

  form:openNextForm → secondOrderModal() - переход к форме контактов

  payment:selected → paymentSelected() - выбор способа оплаты

  address:written → addressWrite() - ввод адреса

  email:written → emailWrite() - ввод email

  phone:written → phoneWrite() - ввод телефона

  cart:openForm → firstOrderModal() - открытие формы заказа

  card:deleted → deleteFromCart() - удаление товара из корзины

  cart:open → cartOpen() - открытие корзины

  modal:close → modalClose() - закрытие модального окна

  card:selected → onCardClick() - выбор карточки товара

  card:action → changeCart() - добавление в корзину

  form:submit → submit() - отправка заказа

  cart:updated → cartUpdate() - обновление корзины

  customer:updated → customerUpdate() - обновление покупателя

  product:updated → renderProducts() - загрузка и отображение товара на странице

  product:currentProductSelected → currentProductSelected() - открытие выбранной карты товара

`createCard(product: IProduct): HTMLElement` - Создает DOM-элемент карточки товара для отображения в галерее.

`onCardClick(id: string): void` - Обрабатывает клик по карточке товара. Открывает модальное окно с подробной информацией о товаре. Если товар уже есть в корзине, кнопка показывает "Удалить из корзины". Если товар бесплатный (цена отсутствует), кнопка блокируется с текстом "Недоступно".



  `changeCart(id: string): void` - Обрабатывает добавление или удаление товара из корзины. Если товар уже в корзине - удаляет его, иначе добавляет. Закрывает модальное окно и обновляет счетчик в шапке.

  `updateHeader(): void` - Обновляет счетчик товаров в шапке сайта на основе текущего размера корзины.

  `modalClose(): void` - Закрывает текущее модальное окно.

  `cartOpen(): void` - Открывает модальное окно с содержимым корзины. Отображает все товары с их ценами и общей стоимостью. Если корзина пуста, кнопка оформления заказа блокируется.

  `deleteFromCart(id: string): void` - Удаляет товар из корзины по идентификатору. Обновляет содержимое корзины и счетчик в шапке.

  `firstOrderModal(): void` - Открывает первый шаг оформления заказа - форму с выбором способа оплаты и вводом адреса. Создает экземпляр формы, если он еще не создан, или переиспользует существующий. Загружает сохраненные данные покупателя и обновляет состояние валидации.

  `paymentSelected(method: TPayment): void` - Обрабатывает выбор способа оплаты. Сохраняет выбранный метод в модели Customer и обновляет форму заказа.

  `addressWrite(address: string): void` - Обрабатывает ввод адреса доставки. Сохраняет адрес в модели Customer и обновляет форму заказа.

  `secondOrderModal(): void` - Открывает второй шаг оформления заказа - форму с вводом контактных данных (email и телефон). Создает экземпляр формы, если он еще не создан, или переиспользует существующий. Загружает сохраненные данные покупателя и обновляет состояние валидации.

  `emailWrite(email: string): void` - Обрабатывает ввод email. Сохраняет email в модели Customer и обновляет форму контактов.

  `phoneWrite(phone: string): void` - Обрабатывает ввод телефона. Сохраняет телефон в модели Customer и обновляет форму контактов.

  `submit(): Promise<void>` - Отправляет заказ на сервер. Собирает данные из модели Customer и корзины, формирует объект заказа. После успешной отправки отображает сообщение об успехе с суммой списанных синапсов и сбрасывает состояние приложения.

  `reset(): void` - Сбрасывает состояние приложения после успешного оформления заказа
